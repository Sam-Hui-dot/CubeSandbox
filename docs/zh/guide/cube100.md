# Cube 100 计划

**Cube 100 · Founding Users Program** —— 首批 100 位 AI Agent 生产先行者案例征集计划。

## 概述

Cube 100 计划面向全球正在或即将使用 CubeSandbox 的企业、团队及组织，征集首批 **100 个**真实、可讲述、可量化的 AI Agent 沙箱生产落地案例。

自 2026 年 4 月开源以来，CubeSandbox 在 80 天内突破 10,000 GitHub Star，发布 11 个版本，汇聚近 70 位全球贡献者，累计提交超 560 个 commits。已有一批用户在真实生产环境中使用 Cube 承载 AI Agent、Coding Assistant、数据分析等业务。

Cube 100 计划旨在将这些实践经验系统化沉淀，为社区提供可参考的架构范例和落地路径，同时为入选团队提供全方位的技术与生态支持。

> 📋 如果你已准备好提交案例，可直接前往[席位申请表单](https://wj.qq.com/s2/27357586/b080/)填写，预计耗时 10–15 分钟。

## 案例内容要求

提交的案例应涵盖以下核心内容：

| 内容模块 | 说明 |
|---------|------|
| **应用阶段** | 当前使用 CubeSandbox 的阶段（生产环境 / POC / 个人项目等） |
| **使用场景** | CubeSandbox 在你的系统中承担的角色（如 Agent 代码执行沙箱、数据分析环境、CI/测试沙箱、Agent 平台底座等） |
| **核心挑战** | 选型 Cube 之前遇到的关键技术或业务问题 |
| **解决方案** | 如何使用 CubeSandbox 解决上述问题，包括关键架构设计与所用特性 |
| **可量化成效** | 至少 1 条可对外披露的量化收益（如冷启动延迟、沙箱密度、基础设施成本、吞吐量等） |
| **部署规模** | 当前 Cube 的部署规模（内部测试 / 小规模生产 / 中大规模生产） |
| **迁移背景**（选填） | 迁移前使用的方案及切换到 Cube 的原因 |

建议同时提供架构图、部署截图或关键指标截图等辅助材料。

## 遴选规则

- 首批限 **100 席**，先提交、先受理、先入选。
- 已提交但未入选的申请，将自动进入下一批候选池，优先纳入下一轮评审。

**遴选维度：**

| 维度 | 说明 |
|------|------|
| 真实性 | 有真实业务场景，非纯 Demo；能提供架构图 / 部署截图 / 关键指标之一 |
| 可讲述性 | 场景清晰、技术选型有思考、能说清"为什么选 Cube" |
| 可量化性 | 有至少 1 项可对外披露的指标（性能 / 成本 / 效率 / 规模任一即可） |
